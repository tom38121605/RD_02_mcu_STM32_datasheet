STM32H503 的 DMA 多通道与主端口关系

1. 硬件架构类比
   DMA 控制器 = 快递公司总部
   DMA 通道 = 快递员（每个 DMA 控制器有 8 个通道，如 GPDMA1_CH0~CH7）
   主端口 = 快递运输路线（每个通道可选择端口 0 或端口 1）
   总线矩阵 = 城市道路网络

   关键点：
      同一路线可被多个快递员复用：多个 DMA 通道可选择相同主端口（如都选端口 0）。
      优先级调度：快递公司（DMA 控制器）通过总线仲裁机制决定谁优先使用路线。

2. 技术细节补充

   在 GPDMA_CxCR 寄存器中：
      LAP 位（第 17 位）：选择通道 x 的主端口（0 或 1）。
      PRI [1:0] 位（第 16:15 位）：设置通道优先级（0~3，0 为最高）。


3. CCR中的LAP 和 CTR1的DAP 和 CTR1的SAP三者的关系     --ok

我们接着用快递员送包裹这个例子来理解。

前面说 CTR1 的 SAP 是快递员从包裹始发地取货时走哪个入口通道，
CTR1 的 DAP 是快递员把包裹送到最终目的地时走哪个到达通道，
而 CCR 中的 LAP 可以想象成快递员在出发去取包裹之前，去获取这次送货任务详细信息（比如包裹清单、送货地址等）的途径选择。


在芯片的 DMA（直接内存访问）机制里，有一个链表寄存器，它就像是一个记录了一系列送货任务的本子。CCR 中的 LAP 就是决定从内存更新这个链表寄存器时，通过哪个 AHB（高级高性能总线）端口来获取更新信息。


举个更形象的例子，假设快递员有一个办公室，办公室有两个门（对应两个 AHB 端口），链表寄存器的更新信息就放在办公室里面。LAP = 0 时，快递员从 “门 0” 进入办公室去获取更新信息；LAP = 1 时，快递员就从 “门 1” 进入办公室获取更新信息。


所以，CTR1 的 SAP 和 CTR1 的 DAP 是关于实际取货和送货过程中通道的选择，而 CCR 中的 LAP 是关于获取送货任务指令信息途径的选择。
